Variable syntax

The syntax for declaring a variable in JavaScript is like this: 

            
              var magicNumber = 42;

Breaking it down

            
var          // is a reserved word used to declare a variable

magicNumber  // is the variable name

=            // is the assignment operator

42           // is the value assigned to the variable

;            // signifies the end of a statement in JavaScript

Arrays

Just like a variable holds a single value, an array holds a collection of values.

Arrays

Think of an array as a bunch of buckets, each of which stores a value.

Arrays

If we were to put the first eight characters of the alphabet into these buckets, it would look like this in JavaScript:

            
var letters = ["a","b","c","d","e","f","g","h"];

An array of the characters "a" through "h" is now stored in the variable `letters`. This is an array of strings.

Arrays

Accessing values

Every element in an array is stored in a specific position known as an index. Arrays are indexed starting at 0 and incrementing by 1 for each position.

 0  1  2  3  4  5  6  7
"a" "b" "c" "d" "e" "f" "g" "h"

Arrays

Accessing values

            
var letters = ["a","b","c","d","e","f","g","h"];
letters[0] // returns "a"
letters[7] // returns "h"

Arrays

Updating values

            
var letters = ["a","b","c","d","e","f","g","h"];
letters[0] = "apples"  // returns "apples"
letters[1] = "bananas" // returns "bananas"
letters      // returns ["apples","bananas","c","d","e","f","g","h"]

Arrays

Inserting values

            
var letters = ["a","b","c","d"];
letters.push("elephant") // returns "elephant"
letters // returns ["a","b","c","d","elephant"]

The push method adds an element to the end of an array.

Arrays

Inserting values

            
var letters = ["a","b","c","d", "elephant"];
letters.unshift("zebra") // returns "zebra"
letters // returns ["zebra","a","b","c","d", "elephant"]

The unshift method adds an element to the start of an array.

What element is in index zero of the array now?

Arrays

Removing values

            
var letters = ["a","b","c","d"];
var char = letters.pop() // returns "d"
letters // returns ["a","b","c"]

The pop method removes an element from the end of an array.

What element is in index zero of the array now?

Arrays

Removing values

            
var letters = ["a","b","c","d"];
var char = letters.shift() // returns "a"
letters // returns ["b","c", "d"]

The shift method removes an element from the start of an array.

What element is in index zero of the array now?

A quick note on Methods

Methods

All data types in JavaScript come with some built in behavior called methods. We've already seen a few array methods with push, pop, shift and unshift.

Method syntax

            
var letters = ["a","b","c"];
letters.push("d");
// --------------------------
letters // the array that we're calling our method on
.       // the dot to signify we're about to call a method
push    // the name of the method
("d")   // the actual calling of the method with a string argument

We call a method with a dot, followed by the method name, followed by a set of parentheses. Sometimes we put other data inside the parens, known as arguments; other times methods don't require arguments.

Properties

Things in JavaScript can also have properties. You access and modify properties similarly to methods, except you do not include the parentheses.

Properties

            
var letters = ["apple","banana","carrot"];
letters.length // returns 3
letters[2].length // returns 6
letters.push("d");
letters.length // returns 4

length is a good example of a property on both arrays and strings. Note how properties don't need parentheses after them.

Methods vs Properties

It can be very confusing to know whether something is a method or an property when you start out with JavaScript. It's something you'll get used to.

Remember you always have the Web to look things up!

Objects

Objects in JavaScript are similar to arrays in that they contain a collection of values.

However, unlike arrays which are ordered by index, the values in objects are stored in keys that you name yourself.

Objects

In our metaphor we give each bucket a name. This is the key. The item inside the bucket is its value. So, a JavaScript object is a collection of key value pairs.

If this collection of buckets was an object, what do you think it would represent?

Objects

Let's instantiate an object in JavaScript:

            
var person = {name: "Sam", age: 28, sex: "male"};

This object contains three key value pairs and the object is stored in the variable `person`.

What data type are the values in this object?

Objects

Accessing values

            
var person = {name: "Sam", age: 28, sex: "male"};
person.name      // returns "Sam"
letters["name"]  // returns "Sam"

Objects

Updating values

            
var person = {name: "Sam", age: 28, sex: "male"};
person.age = 29      // returns 29

person
// now contains
{name: "Sam", age: 29, sex: "male"}

person.age           // returns 29

Objects

Inserting values

            
var person = {name: "Sam", age: 28, sex: "male"};
person.gender = "cis male"  // returns "cis male"

person
// now contains
{name: "Sam", age: 28, sex: "male", gender: "cis male"}

Objects

Removing values

            
var person = {name: "Sam", age: 28, sex: "male", gender: "cis male"}
delete person.sex // returns true

person
// now contains
{name: "Sam", age: 28, gender: "cis male"}

person.sex        // returns undefined

The delete operator removes a key value pair from an object.

Why did I not call `delete` a method? It's relatively rare to use `delete`. Most often you are reading or accessing data. Second most, adding or inserting data. Third most, updating data. Rarely, deleting or removing data.
